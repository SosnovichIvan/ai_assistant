# AI Assistant — План реализации

## Общее описание

AI-ассистент на Go с семантическим поиском по документации, REST API, Telegram ботом и WebSocket для real-time чата.

## Архитектура

              │                       │
    ┌─────────▼──────────┐   ┌────────▼────────┐
    │   HTTP Handlers    │   │  WebSocket Hub  │

| Компонент | Технология |

| Vector DB | PostgreSQL + pgvector |
| ORM | sqlx |
| Telegram Bot | go-telegram-bot-api |

## API контракты

### POST /api/v1/documents
```json
// Request
{
  "title": "Company FAQ",
  "content": "Long text content...",
  "metadata": {
    "source": "manual",
    "category": "support"
  }
}

// Response 201
{
  "id": "uuid",
  "title": "Company FAQ",
  "chunks_count": 15,
  "created_at": "2024-01-15T10:30:00Z"
}
```

### PATCH /api/v1/documents/:id
```json
// Request
{
  "content": "Additional text to append...",
  "append": true
}

// Response 200
{
  "id": "uuid",
  "chunks_count": 18,
  "updated_at": "2024-01-15T11:00:00Z"
}
```

### POST /api/v1/query
```json
// Request
{
  "question": "Как восстановить пароль?",
  "top_k": 5,
  "threshold": 0.7
}

// Response 200 (найден ответ)
{
  "answer": "Для восстановления пароля перейдите в раздел...",
  "sources": [
    {
      "document_id": "uuid",
      "chunk_id": "uuid",
      "text": "Частичный текст чанка...",
      "similarity": 0.89
    }
  ],
  "found": true
}

// Response 200 (ответ не найден)
{
  "answer": null,
  "sources": [],
  "found": false,
  "message": "Не удалось найти релевантную информацию"
}
```

### WebSocket /ws/chat/:room_id
```json
// Client → Server
{ "type": "message", "content": "Привет!", "sender": "user_123" }
{ "type": "typing", "content": "", "sender": "user_123" }
{ "type": "leave", "content": "", "sender": "user_123" }

// Server → Client
{ "type": "message", "content": "Привет! Чем могу помочь?", "sender": "ai", "timestamp": "..." }
{ "type": "typing", "content": "", "sender": "ai" }
```

## Схема БД

```sql
-- Documents
CREATE TABLE documents (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(500) NOT NULL,
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMP DEFAULT NOW(),
    updated_at TIMESTAMP DEFAULT NOW()
);

-- Document chunks with vectors
CREATE TABLE document_chunks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    document_id UUID REFERENCES documents(id) ON DELETE CASCADE,
    content TEXT NOT NULL,
    embedding VECTOR(1536),
    position INT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Messages for chat history
CREATE TABLE messages (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    room_id VARCHAR(100) NOT NULL,
    sender VARCHAR(100) NOT NULL,
    content TEXT NOT NULL,
    message_type VARCHAR(20) DEFAULT 'message',
    created_at TIMESTAMP DEFAULT NOW()
);

-- Telegram users
CREATE TABLE telegram_users (
    id BIGINT PRIMARY KEY,
    username VARCHAR(255),
    is_main_user BOOLEAN DEFAULT FALSE,
    linked_chat_id BIGINT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Create index for vector search
CREATE INDEX ON document_chunks USING ivfflat (embedding vector_cosine_ops);
```
